1. Introduction

Network traffic analysis is a critical task for Security Operations Center (SOC) analysts to detect malicious activities, troubleshoot network issues, and identify security risks.
This project focuses on analyzing live network traffic using Wireshark to understand normal and suspicious behavior across common protocols.

The objective is to simulate real-world SOC scenarios such as:

ICMP reconnaissance

DNS monitoring

HTTP traffic analysis

Credential sniffing

Port scan detection

2. Objective of the Project

Capture and analyze live network traffic

Understand how different protocols behave

Identify potential security threats

Practice SOC-level packet analysis

Gain hands-on experience with Wireshark filters

5. Methodology

6.4 Credential Sniffing (HTTP POST)

Filter Used:

http.request.method == "POST"


Observation:

HTTP POST requests identified

Username and password visible in plaintext (when applicable)

Security Risk:

Credentials transmitted without encryption

Vulnerable to Man-in-the-Middle attacks

Conclusion:
‚ùå Critical security risk identified

6.5 Port Scan / Reconnaissance Detection

Filter Used:

tcp.flags.syn == 1 && tcp.flags.ack == 0


Observation:

Multiple SYN packets sent to different ports

No ACK response from destination

Security Relevance:

Typical behavior of SYN scan

Indicates reconnaissance activity

Conclusion:
‚ö† Potential port scanning detected
